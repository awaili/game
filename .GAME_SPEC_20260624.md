# Slowpapa Game Rewrite Spec (给重写游戏的子代理)

你在为 **https://game.slowpapa.com** 重写一款 H5 小游戏。目标：现代、移动优先、
触控友好、视觉精致、与全站一致。输出**单个自包含 HTML 文件**，覆盖原文件。

## 必须遵守

1. **文件**：写到指定路径 `games/NN_xxx/index.html`，单文件，内联 JS。
2. **共享资源**（同一域名，绝对路径，勿用相对路径）：
   - `<link rel="stylesheet" href="/assets/game.css">`
   - `<script src="/assets/game.js"></script>`（在 `</body>` 前）
3. **HTML 结构**（用 game.css 提供的类，不要自己写顶栏样式）：
   ```html
   <div class="game-shell">
     <div class="game-topbar">
       <a class="back" href="/" aria-label="返回游戏大厅">‹</a>
       <div class="title">游戏名<small>一句话副标题</small></div>
       <div class="stats">
         <div class="stat" data-stat="score"><span class="k">得分</span><span class="v">0</span></div>
       </div>
       <button class="icon-btn" id="help" aria-label="规则">?</button>
     </div>
     <div class="game-stage"><div class="game-board"><!-- 棋盘/canvas --></div></div>
     <!-- 触控控件（移动端必需）：用 .touch-pad / .action-row -->
   </div>
   ```
4. **主题**：页面加载即调用 `SG.applyTheme()`（game.js 已自动执行）；右上角放一个
   主题切换 `icon-btn`，点击 `SG.toggleTheme()`。不要硬编码深色背景——用 CSS 变量。
5. **最高分**：用 `SG.best("<gamekey>", value)` 读写，`<gamekey>` 见下表。
6. **弹窗**：胜负、规则用 `SG.overlay({title, body, score?, buttons})`，不要用 `alert/confirm`。
7. **移动端触控**：方向类游戏必须实现滑动（`SG.swipe(stage, dir => ...)`）+ 屏幕方向键
   `.touch-pad.dir`；按钮类用 `.action-row` 的 `.btn`。键盘也要支持（桌面）。
8. **响应式**：`.game-board` 宽度用 `min(94vw, 460px)`（game.css 已含），canvas 用
   `width:100%`，内部用 CSS 像素绘制并处理 devicePixelRatio 清晰显示。
9. **canvas 清晰度**：按 `window.devicePixelRatio` 放大 backing store，CSS 尺寸固定。
   尺寸变化时重绘。
10. **不要外部依赖**：无 CDN、无外部 JS/CSS、无字体文件、无图片（emoji/CSS/SVG 内联）。
11. **`<head>`**：`charset`、`viewport`（含 `viewport-fit=cover`）、`title`、`meta description`。
    不要引入 GA（首页已有）。
12. **代码质量**：原生 JS（ES2017+），无构建步骤；清晰注释；无 `var` 用 `const/let`；
    防止页面滚动/双击缩放。
13. **玩法打磨**：重新实现要好玩——合理的难度曲线、即时反馈（动画/震动 `navigator.vibrate?.()`）、
    胜负判定、可重开、规则提示。比旧版明显更精致。

## 视觉规范
- 背景由 game.css body 提供（渐变 + 明暗主题），不要给 body 设背景色。
- 棋盘/卡片用 `var(--surface)` + `var(--border)`，圆角 `var(--radius)`。
- 主按钮 `.btn.primary`（渐变），次要 `.btn`。
- 数字用 `font-variant-numeric: tabular-nums`。
- 颜色只用 CSS 变量定义的调色板：accent 紫红、accent-3 青蓝、good/bad/warn。

## 各游戏要求（gamekey 与特殊点）

- `01_2048` key=`2048`：4×4，滑动合并，2048 胜利可继续，最高分持久化，优雅的合并动画
  与新增方块弹出。
- `02_sudoku` key=`sudoku`：9×9 数独，三档难度（简单/中等/困难）`.segment` 选择，点击格子
  选数（底部数字盘 `.action-row` 或滑动），高亮同行/列/宫，冲突标红，可撤销/提示/检查，
  完成计时。预生成或本地生成（生成器 + 挖洞），无需外部题库。
- `03_minesweeper` key=`mine`：扫雷，难度可选（小/中/大），左键挖、长按或右键标旗，
  移动端长按标旗，首次点击安全，自动展开 0 区，胜负判定，计时与剩余雷数。
- `04_nonogram` key=`nonogram`：数织，5×5/10×10 两档，行列提示，点击填充/标记X，
  完成校验，错误提示。canvas 或 DOM 网格均可。
- `05_tetris` key=`tetris`：俄罗斯方块，滑动左右、上滑旋转、下滑软降；7-bag 随机，
  影子投影、消行闪光、分数/等级/行数、暂停、游戏结束。
- `06_sokoban` key=`sokoban`：推箱子，多关内置（至少 6 关，可循环/选关），方向键+滑动，
  撤销一步、重置本关、过关自动下一关，目标点高亮。
- `07_klotski` key=`klotski`：华容道（横刀立马），拖拽或方向滑动移动，步数计数，
  解出曹操出底部即胜，可重置。
- `08_tangram` key=`tangram`：七巧板，拖拽/旋转拼图块到目标轮廓，至少 2 个目标图，
  旋转双击或按钮，完成检测。简化但可玩。
- `09_watermelon` key=`watermelon`：合成大西瓜物理版，从顶部落下水果，相同相碰合成更大，
  简化物理（圆碰撞+重力），溢出顶部判负，分数与最高分，下一个水果预览。
- `10_linkgame` key=`link`：连连看，网格图案，点击两个相同且可折线连通则消除，
  连线高亮动画，洗牌、提示、计时，全部消除即胜。
- `11_watersort` key=`watersort`：倒水排序，点选试管倒入，颜色相同才倒，全管单色即胜，
  可撤销、重置、难度选择。
- `12_gomoku` key=`gomoku`：五子棋，人机对弈（简单 minimax/启发式 AI），可先后手，
  悔棋、重开、胜负判定与连线高亮。棋盘 15×15。
- `13_hanoi` key=`hanoi`：汉诺塔，圆盘数可选 3-7，点击/拖拽柱子移动，非法移动提示，
  步数与最少步数显示，完成即胜。
- `14_memory` key=`memory`：翻牌记忆，4×4/6×6 难度，翻两张配对消除，翻牌动画，
  步数与计时，全配对即胜。
- `15_pin` key=`pin`：见缝插针，旋转的中心轮，点击发射针插入空位，碰已有针判负，
  关卡递增针数，分数最高分。
- `16_idiom` key=`idiom`：成语接龙，内置常用成语库（至少 200 条），给首字玩家选/输入
  下一成语，校验尾字接首字，AI 接龙（库内查找），计分与连胜，可跳过/提示。
  输入用 `<input>`，移动端键盘友好。

## 输出
直接 Write 到指定文件路径，覆盖即可。写完后用一句话回报：游戏名、是否完整可玩、
任何已知限制。**不要**修改其他文件、不要改 nginx、不要碰首页。
